Scheduling tasks with CRON

  • Tech Stack: Linux (Ubuntu), Bash, Python, cron (Scheduled Tasks), SSH (Secure Shell)
  • Project Description: The Automated Weekly Remote Backup project is a demonstration of the power of automation using cron. It focuses on automating the process of securely backing up important data to a remote server on a weekly basis.

Step 1: Backup Script (Already Created)

Step 2: Automate with cron
Edit cron Jobs: Open your crontab file with crontab -e
Schedule the Backup: to run the script every Sunday at midnight:
0 0 * * 0 /path/to/backup_script.sh

Problem Encountered: Password Prompt for Remote Connection

When the backup script was initially run via cron, it encountered a problem. rsync required a password to connect to the remote server, which was not practical for an automated weekly backup process.

Solution: Importing an SSH Key
To resolve the password prompt issue and enable passwordless authentication for rsync, the solution is to use SSH keys

1. Generate SSH Key Pair : ssh-keygen -t rsa -b 4096
2. Copy the Public Key to the Remote Server: ssh-copy-id user@remote-server
3. Ensure Proper File Permissions:
-Check that the .ssh directory on the remote server has the appropriate permissions (700).
-Ensure that the authorized_keys file within the .ssh directory has the correct permissions (600).
4. Test SSH Key Authentication: ssh user@remote-server (no password should be required)
